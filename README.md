# rimworld-mods

Monorepo for [JalapenoLabs](https://www.jalapenolabs.io/) RimWorld mods. All mods live here as
git submodules. Build tooling, CI/CD, and AI-assisted developer workflows are centralised at
the root so individual mod repos stay lean.

---

## Mods

| Mod | Description | Steam |
|-----|-------------|-------|
| [fishing-is-fun](mods/fishing-is-fun) | Makes Odyssey fishing count as recreation and grants a mood buff after a long session | [Workshop](https://steamcommunity.com/sharedfiles/filedetails/?id=3538562620) |

## Prerequisites

| Tool | Version | Purpose |
|------|---------|---------|
| [Git](https://git-scm.com/) | Any | Submodule support |
| [.NET SDK](https://dotnet.microsoft.com/download) | 9.0+ | Compiling mod assemblies |
| [Go](https://go.dev/) | 1.23+ | Running Mage |
| [Mage](https://magefile.org/) | Latest | Build task runner |

Install Mage:
```shell
go install github.com/magefile/mage@latest
```

## Getting started

Clone the repo with all submodules:
```shell
git clone --recurse-submodules git@github.com:JalapenoLabs/rimworld-mods.git
cd rimworld-mods
```

If you already cloned without submodules:
```shell
git submodule update --init --recursive
```

---

## Building

```shell
# Build a single mod
mage build fishing-is-fun

# Build all mods
mage buildAll

# Clean compiled assemblies for a single mod
mage clean fishing-is-fun

# Clean all
mage cleanAll
```

The compiled `.dll` is written to `mods/<mod>/1.6/Assemblies/`.

## Directory layout

```
rimworld-mods/
├── mods/
│   ├── electricity-meter/     # submodule
│   ├── fishing-is-fun/        # submodule
│   └── project-zero-dawn/     # submodule
├── RimworldForCICD/           # submodule — managed DLLs for building
├── binaries/
│   └── 0Harmony.dll           # shared Harmony library
├── .claude/
│   └── skills/
│       ├── test-mod/          # /test-mod skill
│       └── translate-mod/     # /translate-mod skill
├── .github/
│   └── workflows/
│       ├── main.yml           # CI: build + validate on push/PR
│       ├── callable-build-mod.yml  # reusable build workflow
│       ├── release.yml        # manual release packaging
│       └── test-mod.yml       # live RimWorld boot test (self-hosted)
├── legacy/
│   └── publish.ps1            # broken Steam publish script (archived)
├── magefile.go                # build targets
├── go.mod
└── .env.example               # local dev configuration template
```

Each mod follows this internal layout:
```
mods/<mod-name>/
├── About/
│   ├── About.xml              # mod metadata and dependencies
│   └── Preview.png
├── 1.6/
│   ├── Assemblies/            # compiled DLL output (and 0Harmony.dll if needed)
│   ├── Source/                # C# source files
│   ├── Defs/                  # XML definitions
│   └── Languages/             # translations (generated by /translate-mod)
├── Textures/                  # sprites and UI assets
├── mod.csproj                 # .NET project file
└── README.md
```

## CI / CD

| Workflow | Trigger | What it does |
|----------|---------|--------------|
| `main.yml` | push to `main`, pull request | Builds and XML-validates all mods in a matrix |
| `release.yml` | manual dispatch | Builds, packages, and uploads a zip artifact for Steam |
| `test-mod.yml` | manual dispatch | Boots RimWorld on a self-hosted runner and checks the startup log |

CI requires the `SSH_PRIVATE_KEY` repository secret (for submodule access) and
`ANTHROPIC_API_KEY` as an organisation secret (for `test-mod.yml`).

## Local development

Copy `.env.example` to `.env` and fill in your machine's paths:

```shell
cp .env.example .env
```

```dotenv
RIMWORLD_DIR=E:\SteamLibrary\steamapps\common\RimWorld
RIMWORLD_LOCAL_LOW=C:\Users\YourName\AppData\LocalLow\Ludeon Studios\RimWorld by Ludeon Studios
```

### Claude Code skills

With a `.env` in place, two slash commands are available inside Claude Code:

**`/test-mod <mod-name>`**
Builds the mod, temporarily swaps `ModsConfig.xml` to activate only that mod, boots
RimWorld, captures the startup log, and restores everything on exit. Requires a live
RimWorld install and a desktop session.

```
/test-mod fishing-is-fun
```

**`/translate-mod <mod-name>`**
Translates all `Defs/` XML files to all 29 RimWorld-supported languages. Existing
translation files are respected — only missing entries are added.

```
/translate-mod fishing-is-fun
```

## Releasing a mod

Trigger the **Release Mod** workflow from the GitHub Actions tab. Provide the mod name
and RimWorld version. The workflow builds, validates, and uploads a ready-to-ship zip.

---

## Adding a new mod

1. Add the mod repo as a submodule under `mods/`:
   ```shell
   git submodule add git@github.com:JalapenoLabs/rimworld-<mod>.git mods/<mod>
   ```
2. Ensure `mods/<mod>/mod.csproj` exists at the mod root (Mage discovers mods by this file).
3. Add the mod to the matrix in `.github/workflows/main.yml`.
4. Commit and push.
